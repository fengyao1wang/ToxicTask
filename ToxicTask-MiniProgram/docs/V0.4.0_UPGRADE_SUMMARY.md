# ToxicTask v0.4.0 升级总结

> 更新日期：2026-04-05
> 版本：v0.4.0 (社交契约升级版)

## 核心升级内容

### 1. 任务创建流程重构（两步走）

**问题**：原版本将"好友监督"作为任务创建页底部的开关，入口过深，用户感知不强。

**解决方案**：
- **第一步 (`pages/tasks/create.tsx`)**：设置任务基础信息
  - 任务标题
  - 任务类型（单次/重复）
  - 押注金额
  - 完成时限
  - 按钮文案改为："下一步：设置契约"

- **第二步 (`pages/tasks/contract.tsx`)**：社交契约设置页
  - 红黑警示风格设计（渐变背景 + 红色边框）
  - 监督开关（默认关闭）
  - 监督赏金设置（5-50币，快捷选项 + 滑块）
  - 失败记录可见度选择：
    - 好友圈（默认）
    - 全服围观（成功额外+1币激励）
  - 主CTA："邀请微信好友监督"（带动态文案）
  - 次级操作："不找监督，直接创建"

**产品逻辑**：
- 强化监督功能的仪式感和严肃性
- 通过红黑警示设计强调风险
- 降低用户心理门槛（可选择可见度）
- 提供激励机制（全服围观+1币）

---

### 2. 独立社交互动Tab

**新增页面**：`pages/social/index.tsx`

**功能模块**：

#### 2.1 待我裁决（作为监督者）
- 展示所有需要审核的任务（`supervision_status === 'evidence_submitted'`）
- 卡片内容：
  - 任务标题 + 押注金额
  - 证据图片（可点击预览）
  - 证据文字说明
  - 审核按钮：通过（绿色）/ 拒绝（灰色）
  - 奖励提示：通过获得X币 | 拒绝获得Y币
- 审核操作：
  - 点击按钮弹出评论输入框
  - 通过：鼓励性评论
  - 拒绝：嘲讽性评论（不带脏字）
  - 评论会记录到任务的 `supervisor_comment` 字段

#### 2.2 我的契约（作为发起者）
- 展示所有自己发起的监督任务
- 卡片内容：
  - 任务标题 + 状态标签（等待接受/进行中/待审核/已通过/已拒绝）
  - 押注金额 + 赏金金额
  - 监督者评论（如果有）
  - 点击跳转到任务详情页

**入口位置**：
- 个人中心页面 → "获取尊严币" 区域 → "社交互动"卡片
- 图标：🤝
- 说明：监督好友

---

### 3. 耻辱墙可见度分级系统

**数据模型扩展**：
- `Task` 接口新增：
  - `visibility: 'private' | 'friends' | 'public'` - 失败记录可见度
  - `supervisor_comment?: string | null` - 监督者评论
- `ShameLog` 接口新增：
  - `visibility` - 继承自任务的可见度设置
  - `supervisor_comment` - 监督者评论

**UI 升级**：
- 新增 Tab 切换：
  - **好友圈**：显示 `visibility === 'friends'` 的记录
  - **全服围观**：显示 `visibility === 'public'` 的记录
- 耻辱卡片新增：
  - 监督者评论区域（红色边框高亮）
  - 评论标签："监督者评论："

**产品逻辑**：
- 降低用户心理门槛（默认仅好友可见）
- 通过激励机制（+1币）引导用户选择公开
- 增强社交压力（监督者评论）

---

### 4. 数据流更新

#### 4.1 appStore 扩展
新增方法：
```typescript
createShameLog(
  taskId: string,
  userId: string,
  taskTitle: string,
  betAmount: number,
  visibility?: 'private' | 'friends' | 'public',
  supervisorComment?: string
): Promise<void>

getShameLogsByVisibility(
  visibility: 'friends' | 'public'
): ShameLog[]
```

#### 4.2 taskStore 更新
- `createTask`：支持 `visibility` 字段（默认 'friends'）
- `reviewEvidence`：审核拒绝时传递 `visibility` 和 `supervisor_comment` 到耻辱记录
