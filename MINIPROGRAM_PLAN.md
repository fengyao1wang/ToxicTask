# ToxicTask 微信小程序开发计划

## 为什么转向小程序？

1. ✅ **完全免费** - 无需支付 Apple Developer $99/年
2. ✅ **无需下载** - 用户在微信内直接使用
3. ✅ **快速验证** - 更适合测试产品和用户体验
4. ✅ **中国市场** - 微信用户覆盖率极高

## 技术栈选择

### 方案对比

| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **Taro + React** | React 语法，学习成本低，可复用现有逻辑 | 需要重写 UI | ⭐⭐⭐⭐⭐ |
| uni-app + Vue | 一套代码多端运行 | 需要学习 Vue | ⭐⭐⭐ |
| 原生小程序 | 性能最好 | 学习成本高，代码不可复用 | ⭐⭐ |

### 推荐：Taro + React

**理由：**
- 你已经熟悉 React 和 TypeScript
- 可以复用现有的业务逻辑（Zustand store、Supabase API 调用）
- 未来可以编译成多端（小程序、H5、APP）

## 可复用的代码

从现有 React Native 项目中可以直接复用：

1. **业务逻辑层（100% 复用）**
   - `lib/supabase/` - Supabase API 调用
   - `lib/stores/` - Zustand 状态管理
   - `lib/services/` - AI 服务
   - `types/` - TypeScript 类型定义

2. **需要重写的部分**
   - UI 组件（React Native → Taro 小程序组件）
   - 路由（Expo Router → Taro 路由）
   - 样式（NativeWind → Taro 样式）

## 开发步骤

### 第一阶段：环境搭建（1 天）

1. 注册微信小程序账号（免费）
2. 安装微信开发者工具
3. 创建 Taro 项目
4. 配置 TypeScript + Zustand

### 第二阶段：迁移核心功能（3-5 天）

1. 迁移 Supabase 配置和 API
2. 迁移状态管理（Zustand）
3. 重写 UI 组件：
   - 登录/注册页面
   - 任务列表页面
   - 创建任务页面
   - 个人资料页面
   - 耻辱墙页面

### 第三阶段：测试和优化（2-3 天）

1. 功能测试
2. UI/UX 优化
3. 性能优化
4. 提交审核

## 预计时间

- **总开发时间：** 1-2 周
- **审核时间：** 1-3 天

## 下一步行动

1. 注册微信小程序账号
2. 创建新的 Taro 项目目录
3. 开始迁移代码

## 注意事项

- 小程序有包大小限制（主包 2MB，总包 20MB）
- 需要配置服务器域名白名单（Supabase 域名）
- AI API 调用需要配置合法域名
- 小程序不支持某些 Web API，需要使用小程序专用 API
